# Trel

Лёгкий лаунчер Minecraft на Electron + React + TypeScript. Один `.exe`, без рекламы, с автообновлениями и аккуратной работой с мод-загрузчиками.

## Возможности

### Версии Minecraft
- Каталог всех версий с `launchermeta.mojang.com`: релизы, снапшоты, old beta, old alpha — вплоть до самых первых rd-/Classic-версий.
- Автоматическое скачивание клиента, библиотек, ассетов и нативов через `@xmcl/core` + `@xmcl/installer`.
- Авто-обновление списка версий каждые 30 минут — новые снапшоты появляются без перезапуска.
- Описания исторических версий (что нового было в `1.13`, `12w27a` и т.п.).

### Мод-загрузчики
- Поддержка **Fabric**, **Quilt**, **Forge**, **NeoForge** через официальные установщики.
- После установки лоадер «впитывает» родительскую ваниль — на диске остаётся **одна** папка вместо двух (`versions/<loader>/` без `versions/<vanilla>/` рядом).
- Кнопка **«Вернуть к ванили»** — удаляет лоадер, переустанавливает чистую ваниль, переносит миры/моды/паки обратно.

### Контент
- Моды, шейдеры, ресурс-паки и текстур-паки — у **каждой версии свои**, не смешиваются.
- Физически файлы лежат в `versions/<id>/mods/`, при запуске прокидываются в `gameDir/mods/` через NTFS junction. Игра ничего не замечает.
- В UI: добавление через диалог выбора файла, удаление, включение/отключение модов (через `.disabled`), быстрый переход к папке.
- Переключение между установленными версиями прямо со страницы контента.

### Аккаунты
- **Только офлайн-режим** (гостевые профили). UUID детерминирован от никнейма по схеме vanilla offline-mode — тот же ник = тот же UUID и сохранения.
- Несколько профилей одновременно, быстрое переключение.
- Работает в одиночной игре и на серверах с `online-mode=false`. **Microsoft-аккаунтов нет** — для официальных серверов нужно использовать другой лаунчер.

### Java
- Автодетект всех установленных JDK/JRE на Windows (Microsoft Store, AdoptOpenJDK, Eclipse Adoptium, Oracle, Liberica и т.п.).
- При отсутствии нужного мажора — автоматическое скачивание Eclipse Temurin JRE (Adoptium) под нужную версию (8, 17, 21).
- Подбор Java под версию игры: 8 для старых, 17 для 1.18+, 21 для 1.20.5+.
- Возможность вручную указать свой путь к `java.exe`.

### Миры и сохранения
- Список миров из `saves/`, включая legacy-варианты с loose `level.dat` (Pre-Classic / rd-*).
- Бэкап в zip, удаление, открытие папки.
- Корректная работа со старыми форматами хранилища, где сохранения были в `%APPDATA%\.minecraft`.

### Автообновления
- `electron-updater` + GitHub Releases. Дифференциальные апдейты через `blockmap` — пользователь качает только дельту, не весь `.exe` заново.
- Уведомление в углу, ручная кнопка «Проверить» в настройках.

### Прочее
- Кастомный title bar, тёмная тема, плавные анимации, русский интерфейс.
- Сброс лаунчера (с сохранением или без) и встроенный uninstall.
- Логи запуска игры прямо в окне лаунчера (можно скрыть/очистить).

## Стек

- **Electron 33** — кроссплатформенный shell.
- **React 18 + TypeScript 5** — UI.
- **Vite** — сборка renderer.
- **electron-builder** — упаковка в NSIS-установщик и portable-сборку.
- **@xmcl/core**, **@xmcl/installer** — работа с Minecraft (parsing, install, launch).
- **axios** — сетевые запросы.

## Требования

- **Node.js 22+** (для разработки и сборки).
- **Windows 10/11** для сборки `.exe` (electron-builder поддерживает кросс-сборку, но протестировано на Windows).
- Java для запуска **самой игры** установлена не нужна — лаунчер качает её сам, если нужного мажора не нашёл.

## Разработка

```powershell
npm install
npm run build:electron        # один раз компилирует main/preload в dist-electron/
npm run dev                   # запускает Vite на localhost:5173
npm start                     # в другой консоли — Electron, подцепится к Vite
```

При работе над main-процессом нужно перезапускать `npm run build:electron` и `npm start`. Renderer обновляется в Vite автоматически (HMR).

## Сборка релиза

```powershell
npm run dist
```

В `release/` появятся:
- `Trel-<version>-x64.exe` — NSIS-установщик.
- `Trel.exe` — portable (запускается без установки).
- `latest.yml` + `Trel-<version>-x64.exe.blockmap` — для авто-обновлений.

Иконка `build/icon.ico` генерируется скриптом `scripts/build-icon.js` (без зависимостей, рисует pixel-by-pixel и пакует в multi-res ICO).

## Публикация на GitHub

Не автоматическая — workflow только проверяет что проект собирается. Для релиза:

```powershell
.\scripts\publish.ps1 -Token ghp_xxxxxxxxxxxx
```

Скрипт удаляет все существующие релизы и теги в репозитории, создаёт новый по `version` из `package.json` и заливает 4 файла из `release/`. Токен — Personal Access Token со scope `repo`.

## Структура

```
src/
  main/         Electron main process — окно, IPC, установка/запуск MC, лоадеры, обновления
  preload/      contextBridge API для renderer (типизированный)
  renderer/     React-приложение
  shared/       Типы, общие для main и renderer
build/          Иконки (генерируются)
scripts/        Утилиты сборки и публикации
```

## Хранилище данных

- `%APPDATA%\Trel\` — настройки лаунчера, аккаунты, скачанная Java.
- `%APPDATA%\Trel\minecraft\` (по умолчанию) — gameDir: версии, библиотеки, ассеты, миры. Путь меняется в настройках.

При апгрейде с старой версии (под именем `AuroraLauncher`) данные мигрируют автоматически при первом запуске.

## Важно

Minecraft — собственность Mojang/Microsoft. Лаунчер не предоставляет игру и не обходит лицензию: он скачивает официальные клиенты с серверов Mojang, но **запускать их без купленного аккаунта** разрешено только в одиночной игре и на серверах с `online-mode=false`. Игра на официальных серверах требует Microsoft-аккаунта, которого Trel не поддерживает.
